# TFTGuider 需求文档（PRD v0.1）

## 1. 背景与目标

### 1.1 背景
TFT 玩家在赛后复盘时，常常需要在多个页面来回查看战绩、阵容、环境信息，难以在短时间内形成可执行改进结论。

### 1.2 产品目标
TFTGuider 旨在通过单页分析体验，聚合玩家近 30 场对局的关键信息，输出直观趋势与可执行建议，帮助玩家快速复盘并调整后续对局策略。

### 1.3 非目标（当前阶段）
- 不构建完整战绩社区或排行榜系统。
- 不实现 Riot OAuth 登录全链路。
- 不在当前阶段承诺真实录像回放能力。

---

## 2. 用户与场景

### 2.1 目标用户
- 想提升上分效率的 TFT 排位玩家。
- 需要快速了解近期状态波动的中高频玩家。

### 2.2 核心使用场景
1. 用户输入 `region + gameName + tagLine` 后，查看最近 30 场综合表现。
2. 用户识别近期名次/稳定性波动，定位问题阶段。
3. 用户查看常用阵容骨架与核心英雄，判断是否过度单一。
4. 用户参考系统复盘建议，形成下一阶段改进方向。

---

## 3. 功能需求

### 3.1 数据接入
- 支持根据 Riot ID 查询玩家信息与近 30 场 TFT 对局。
- 支持区服参数校验（如 KR、JP1、NA1、EUW1 等）。
- 当 Riot API 不可用时，自动降级使用本地演示数据，确保页面可展示。

### 3.2 顶部概览
- 展示玩家基础信息：昵称、区服、段位。
- 展示核心指标：平均名次、前四率、吃鸡率、波动指数。
- 展示时间窗口对比：最近 10 场 vs 前 20 场。

### 3.3 趋势分析
- 名次趋势折线图。
- LP/替代指标趋势图（若真实 LP 不可用需标注说明）。
- 名次分布可视化。

### 3.4 阵容与英雄分析
- 高频羁绊骨架统计与卡片展示。
- 核心英雄使用频次统计。
- 英雄协同关系热力图。
- 阵容数据表、阵容使用率环图、阵容分布气泡图。

### 3.5 风格诊断与建议
- 输出经济倾向、运营倾向、稳定性标签。
- 提供与环境热门阵容的对照信息。
- 生成可执行复盘建议（例如：节奏优化、阵容池扩展）。

### 3.6 最近对局复盘
- 列表展示最近每一局核心信息（名次、时间、等级、羁绊、关键单位）。
- 支持逐局展开查看。
- 保留“查看录像”入口占位（可见但不承诺可用）。

---

## 4. 数据与规则要求

### 4.1 官方数据字段
系统应尽可能使用 Riot 官方可获得字段，包括但不限于：
- 段位与 LP 快照。
- 对局基础信息与玩家对局明细。
- 最终单位、羁绊、强化符文、淘汰人数、金币等。

### 4.2 非官方可得字段处理
对于 Riot 官方不可直接提供的字段（如单局 LP 变化、goldTrend、levelUpTimeline）：
- 必须明确标识为不可得或推断字段。
- 禁止伪造“官方真实数据”的表达。

### 4.3 推断策略
可基于最近 30 场进行统计与规则推断，产出：
- 羁绊骨架命名。
- 核心英雄频率。
- 风格标签与建议。
- 与环境热门阵容的相似度结果。

---

## 5. 非功能需求

### 5.1 性能
- 在常规网络条件下，单次分析应在可接受时间内返回并渲染（目标 < 5s，受第三方接口影响可浮动）。

### 5.2 可用性
- 当外部数据源失败时，系统应给出明确提示并保证页面主体可用。
- 错误信息应可读，避免仅输出技术异常堆栈。

### 5.3 可维护性
- 前后端代码结构清晰，关键数据处理逻辑应可定位。
- 核心字段与推断规则应可扩展，便于后续接入新版本机制。

### 5.4 合规性
- API Key 通过环境变量配置，不硬编码。
- 不要求用户输入敏感账号凭据。

---

## 6. 约束与依赖

### 6.1 外部依赖
- Riot TFT API。
- MetaBot.GG 热门阵容页面（作为环境参考层）。

### 6.2 已知约束
- 环境基准依赖第三方页面结构，存在变更风险。
- 当前版本不支持真实录像回放。
- 当前主要以原生前端 + Node.js 简化实现为主。

---

## 7. 验收标准（MVP）

满足以下条件可视为 MVP 达成：
1. 用户可基于 Riot ID 完成一次近 30 场分析。
2. 页面可展示概览、趋势、阵容/英雄分析、风格诊断、逐局复盘五大模块。
3. 对“官方不可得字段”有明确标识，文案无误导。
4. Riot API 不可用时可自动回退演示数据并继续展示。
5. 关键接口 `/api/tft/analyze` 行为稳定，异常路径可被提示。

---

## 8. 里程碑建议

- **M1：分析看板 MVP（当前）**
  - 完成数据接入、单页展示、基础推断与降级。
- **M2：标准阵容识别增强**
  - 阵容识别与命名更标准化，引入更完整映射。
- **M3：节奏分析增强**
  - 引入更细粒度阶段节奏指标与对局对比。
- **M4：账号体系与回放能力**
  - 评估接入登录态与真实录像回放链路。
